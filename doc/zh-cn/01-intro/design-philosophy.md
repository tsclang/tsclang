# 设计理念

---

在每一项设计决策中，TSClang 都遵循严格的优先级层次：

## 三个优先级

1. **内存安全** —— 所有权、借用检查器、无 GC
2. **性能和类型** —— 零成本抽象、严格类型
3. **TS 语法** —— 尽可能保留，但不能以牺牲 #1 和 #2 为代价

目标不是"现有 TS 代码无需改动即可编译"，而是"TS 开发者能识别语法并感到熟悉"。

## TS 语法优先

只有当 TS 没有合适的构造时，才借用 Rust、C、Go 的语法。

新概念通过 TS 兼容的语法嵌入：

| 概念 | Rust | TSClang |
|------|------|---------|
| 不可变借用 | `&T` | `Ref<T>` |
| 可变借用 | `&mut T` | `Mut<T>` |
| 可变变量 | `let mut` | `let mut` |
| 只读 | `let`（默认） | `const` / `readonly` |

类被保留了下来，尽管 Rust 中没有 —— 它们在 TS 中存在，开发者很熟悉。

## 每个决策的问题

> *这能否通过现有的 TS 语法或其自然扩展来表达？*

如果可以 —— 使用 TS 语法。如果不可以 —— 找到不冲突的最小扩展。

## 向后兼容

简单的原生 TS 代码（不含外部库）应当可以编译，或只需少量简单修复且仍保持有效的 TS：

```typescript
let a = 10          // 可能需要显式注解
let a: number = 10  // 在 TS 和 TSClang 中都有效
```

包含类、对象、数组、循环、模板字面量的代码 —— 可以开箱即用或只需少量修改。
