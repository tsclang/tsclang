# Приоритет операторов

[← Вверх](./index.md) | [Предыдущий ←](./optional.md)

---

Таблица приоритетов операторов TSClang — от высшего (выполняется первым) к низшему. Операторы на одном уровне имеют указанную ассоциативность.

## Таблица приоритетов

| Приоритет | Оператор(ы) | Ассоциативность | Описание |
|-----------|-------------|-----------------|----------|
| 18 | `()` | — | Группировка |
| 17 | `.` `?.` `[]` `()` | Левая | Доступ к члену, optional chaining, индексация, вызов |
| 16 | `++` `--` | — | Постфиксный инкремент / декремент |
| 15 | `!` `~` `+` `-` `++` `--` | Правая | Унарные: НЕ, побитовое НЕ, унарный плюс/минус, префиксный инкремент/декремент |
| 14 | `**` | Правая | Возведение в степень |
| 13 | `*` `/` `%` | Левая | Умножение, деление, остаток |
| 12 | `+` `-` | Левая | Сложение, вычитание |
| 11 | `<<` `>>` `>>>` | Левая | Битовые сдвиги |
| 10 | `<` `<=` `>` `>=` | Левая | Сравнение порядка |
| 9 | `==` `!=` `===` `!==` | Левая | Равенство / неравенство |
| 8 | `&` | Левая | Побитовое И |
| 7 | `^` | Левая | Побитовое XOR |
| 6 | `\|` | Левая | Побитовое ИЛИ |
| 5 | `&&` | Левая | Логическое И |
| 4 | `\|\|` `??` | Левая | Логическое ИЛИ, nullish coalescing |
| 3 | `? :` | Правая | Тернарный оператор |
| 2 | `=` `+=` `-=` `*=` `/=` `%=` `**=` `&=` `\|=` `^=` `<<=` `>>=` `>>>=` `&&=` `\|\|=` `??=` | Правая | Присваивание |
| 1 | `,` | Левая | Запятая (в списках аргументов) |

## Примеры

### Арифметика

```typescript
2 + 3 * 4              // 14 — * выше +, поэтому 2 + (3 * 4)
2 ** 3 ** 2            // 512 — ** правоассоциативен: 2 ** (3 ** 2)
```

### Логические и побитовые

`&`, `^`, `|` — **разные** приоритеты (8, 7, 6). Это важно при комбинировании:

```typescript
a | b & c              // a | (b & c) — & выше чем |
a ^ b | c              // (a ^ b) | c — ^ выше чем |

// сравнения выше побитовых:
a & b == c             // a & (b == c) — == выше чем &
// для побитового И между сравнениями — скобки:
(a == b) & (c == d)    // ok — явные скобки
```

### Присваивание

Присваивание — самый низкий приоритет (после `,`), правоассоциативный:

```typescript
a = b = c = 5          // a = (b = (c = 5)) — все получают 5
```

### Тернарный оператор

Правоассоциативный — вложенные тернарные группируются справа:

```typescript
x ? a : y ? b : c      // x ? a : (y ? b : c)
```

## Особенности

### `??` нельзя смешивать с `||` / `&&`

Операторы `||` и `??` находятся на одном уровне приоритета (4), но их смешивание без скобок — ошибка компилятора:

```typescript
a || b ?? c            // error: mixing || and ?? requires parentheses
(a || b) ?? c          // ok
a || (b ?? c)          // ok
```

### Доступ к члену (`.`) — высший приоритет

```typescript
obj.method().field[0]  // ((obj.method()).field)[0]
```

### Сдвиги ниже аддитивных операций

```typescript
1 << 2 + 3             // 1 << (2 + 3) = 1 << 5 = 32, не (1 << 2) + 3 = 7
```
