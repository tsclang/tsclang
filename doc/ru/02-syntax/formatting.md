# Форматирование

[← Вверх](./index.md) | [Следующий →](./truthy-falsy.md)]

Форматирование кода — это область ответственности **отдельного инструмента**, а не компилятора. Компилятор (`tsclang build`) проверяет только семантику и полностью игнорирует стиль кода. Это позволяет серверу языка (автодополнение, типы, рефакторинг) корректно работать даже тогда, когда разработчик пишет незаконченный или неформатированный код.

Инструменты и их роли:

| Инструмент | Роль |
|---|---|
| `tsclang build` | Семантические ошибки, форматирование **игнорируется** |
| `tsclang lint` | Семантические ошибки + предупреждения о стиле, CI-проверка — exit code 1 при нарушениях |
| `tsclang lint --fix` | Форматирует код на месте (аналог `prettier` / `gofmt`) |
| IDE | Format-on-save через плагин |

```bash
tsclang lint          # check without changes — for CI
tsclang lint --fix    # format code in place
```

TSC следует соглашениям TypeScript/JavaScript.

---

## Точка с запятой

Точка с запятой **необязательна**. Действует механизм ASI (Automatic Semicolon Insertion), как в JavaScript:

```typescript
const x = 1       // without ;
const y = 2;      // with ; — also ok
```

## Фигурные скобки

Рекомендуется использовать фигурные скобки для **всех** блоков:

```typescript
if (x > 0) {           // recommended
    doSomething()
}

if (x > 0) doSomething()   // allowed (linter may warn)
```

### Стиль расположения открывающей скобки

Рекомендуется стиль K&R (скобка на той же строке):

```typescript
function foo(): void {   // recommended
}

function bar(): void     // allowed (linter may warn)
{
}
```

## Отступы

Отступы **не влияют** на семантику (в отличие от Python). Рекомендуется 4 или 2 пробела; табы допустимы.

## Кавычки

Одинарные и двойные кавычки эквивалентны. Шаблонные строки поддерживают интерполяцию:

```typescript
const a = "hello"
const b = 'hello'             // same
const c = `Hello, ${name}!`   // template literal
```

## Висячая запятая

Висячая запятая (trailing comma) допустима **везде**:

```typescript
const obj = { a: 1, b: 2, }        // ok
function foo(x: i32, y: i32,) {}    // ok
const arr = [1, 2, 3,]              // ok
```

## Переносы строк

Разрывы строк допустимы после:

- `,` — запятая
- `(` — открывающая скобка вызова
- `{` — открывающая фигурная скобка
- бинарного оператора (`+`, `&&`, `||`, …)

## Комментарии

Поддерживаются три вида комментариев:

```typescript
// single-line
/* multi-line */
/** JSDoc */
```

## Пробелы вокруг операторов

Пробелы вокруг операторов рекомендуются. Компилятор не проверяет.

## Пробелы в аннотациях типов

Рекомендуется пробел **после** двоеточия, но не до:

```typescript
const x: i32 = 5          // recommended
const x :i32 = 5          // allowed (linter may warn)
```

## Дженерики

Внутри угловых скобок `< >` пробелы **не рекомендуются**:

```typescript
function first<T>(arr: T[]): T { /* ... */ }   // recommended
function first< T >(arr: T[]): T { /* ... */ } // linter will warn
```

## Union-типы

Пробелы вокруг `|` рекомендуются:

```typescript
type Result = i32 | null      // recommended
type Result = i32|null        // linter may warn
```

## Стрелочные функции

Скобки вокруг параметра **обязательны**, если есть аннотации типа. Без аннотаций — опциональны:

```typescript
const f = (x: i32): i32 => x + 1     // annotations → parens required
const f = x => x + 1                  // no annotations → parens optional
```

## Цепочки вызовов

Если цепочка методов длинная, каждый вызов размещается на отдельной строке:

```typescript
const result = items
    .filter(x => x > 0)
    .map(x => x * 2)
    .reduce((a, b) => a + b, 0)
```

## Тернарный оператор

Простые выражения — в одну строку. Сложные — в несколько строк:

```typescript
// simple — inline
const label = count > 0 ? "items" : "no items"

// complex — multiline
const label = count > 1000
    ? "many items"
    : count > 0
        ? "some items"
        : "no items"
```

## Пустые строки

- **Одна** пустая строка между функциями / классами на верхнем уровне.
- **Максимум одна** пустая строка внутри тела функции.
- **Без пустых строк** сразу после `{` или перед `}`.

## Длина строки

Рекомендуемый максимум — **120 символов**. Компилятор не принуждает, но `tsclang lint` может выдавать предупреждения.

## Конец файла

Файл должен заканчиваться **одним символом новой строки** (`\n`).
