# Что такое TSClang

[← Вверх](./index.md) | [Следующий →](./design-philosophy.md)

---

TSClang — компилируемый язык с синтаксисом TypeScript, который транслирует `.tsc`-файлы в читаемый C-код и автоматически генерирует `CMakeLists.txt`.

## Зачем

Много разработчиков идут с TypeScript в C — и это боль. У C нет достойной экосистемы: нет пакетного менеджера, нет удобной кросс-компиляции, нет встроенных проверок безопасности памяти.

TSClang решает это:

- **Знакомый синтаксис** — TS-разработчик узнаёт конструкции и сразу продуктивен
- **Безопасная память** — ownership и borrow checker на этапе компиляции, без GC
- **Единая экосистема** — зависимости, кросс-компиляция, сборка из коробки
- **Читаемый C-выхлоп** — можно инспектировать, отлаживать, комбинировать с ручным C

## Для чего

**Сейчас:**

- Серверный код — HTTP, сокеты, бэкенд
- Десктоп — CLI/TUI, файловые менеджеры, офисные приложения

**Важно:**

- Системный уровень — драйверы, ОС
- Embedded — Arduino, ESP, Raspberry Pi
- Игры — через OpenGL, DirectX

**Мечта:**

- Кросс-платформа — Windows, Linux, Mac, Android, iOS
- Ретро-платформы — ZX Spectrum, NES, Sega, MS-DOS

## Файловое расширение

`.tsc` — исходный файл TSClang.

```typescript
// hello.tsc
console.log("Hello world")
```

Компилируется в:

```c
// hello.c
#include "runtime.h"
int main(void) {
    tsc_console_log(tsc_string_from_cstr("Hello world"));
    return 0;
}
```
