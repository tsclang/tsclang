# Миграция: TypeScript → TSClang

[← Вверх](../index.md) | [Следующий →](./automatic.md)

---

Руководство для разработчиков, переходящих с TypeScript на TSClang. Описывает автоматические и ручные преобразования, несовместимые паттерны и новые возможности.

## Обзор процесса

TSClang стремится к максимальной совместимости с TypeScript-синтаксисом. Большинство TypeScript-кода переносится без изменений или с минимальными правками. Процесс миграции делится на три этапа:

1. **Автоматические правки** — `tsclang migrate` применяет механические трансформации
2. **Ручные правки** — паттерны, которые невозможно автоматизировать безопасно
3. **Несовместимые паттерны** — конструкции без прямого аналога, требующие перепроектирования

## Быстрая проверка

```bash
tsclang migrate ./src            # dry-run: показать что изменится
tsclang migrate ./src --fix      # применить автоматические правки
tsclang migrate ./src --check    # CI: exit 1 если есть несовместимости
```

## Что переносится без изменений

Интерфейсы, функции с типами, стрелочные функции, классы (без `extends`), дженерики, `try/catch`, шаблонные строки, деструктуризация — всё это работает как в TypeScript. Подробнее — в разделе [Ручная миграция](./manual.md).

## Ошибки

| Ошибка | Причина |
|--------|---------|
| `undefined is not defined` | Используется `undefined` — заменить на `null` |
| `throw requires Error instance` | Бросается строка или число — обернуть в `new Error()` |
| `export default is not supported` | Заменить на именованный экспорт |
| `extends is not supported` | Классовое наследование — заменить на композицию |
